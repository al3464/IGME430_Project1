Project1 Documentation

Relative URL - All endpoints return JSON responses.

GET: http://127.0.0.1:3000/getCats

The endpoint returns a JSON object with a cats property.

Example: { "Creamy": { "name": "Creamy", "age": 6, "catBirth": "2019-4-11", "breedValue": "/ragdoll", "breedText": "Ragdoll" } }

Example query GET: http://127.0.0.1:3000/getCats?age=11

The query params is age. The example url can filter cat's information by specific age.

POST: /addCat

Add a new cat. Sucess input will return 201 and create a cat card. If any user input is empty will return 400 bad request.

POST: /updateCat

If the user add a repeat user name will trigger updateCat. updateCat will update an exsiting data and return a 204 no content to remind user this name is already exist. Will not create a new cat card.

Query Params

Query params: age Type: number Description: cat's information could filtered by age.

Body Params

Parameter: name Type: string Description: cat's name

Parameter: age Type: number Description: cat's age

Parameter: catBirth Type: string Description: cat's date of birth

Parameter: breedValue Type: string Description: example:/ragdoll to load cat's image

Parameter: breedText Type: string Description: cat's breed

How To Use

Simply fill in the name, birthday, and age, select the cat’s breed and press add, you’ll receive a cat card. It will also remind you how many days are left until your cat’s next birthday—or if it has already passed this year.
